BtDeck是为使用多个bt客户端的pter提供BitTorrent客户端(qBittorrent、Transmission)的统一管理界面。

**该项目的绝大部分功能都是按照我个人使用习惯产生的需求进行开发，希望也能解决各位的痛点。**

核心功能：

- **统一数据中枢**
  BtDeck 本地管理持久化所有下载器的种子核心数据
  在 BtDeck Web 界面操作后，实时同步保持数据始终一致
- **下载器集中管理 + 智能限速**
  统一查看/编辑所有下载器的速度、标签、状态
  支持多条 Cron 定时限速规则（早高峰限上传、深夜全速等）
- **多维度组合搜索**
  支持按种子名称、大小、路径、分享率、Tracker URL、Tracker 状态、Tracker 消息等多种条件灵活查询
- **一键痛点排查**
  跨下载器重复种子检测
  Tracker 报错种子快速定位
- **全统一操作面板**
  开始 / 暂停 / 强制校验 / Tracker 更新 / 种子转移 / 修改保存路径
  无需频繁切换不同下载器界面
- **全局一键替换 Tracker**
  一次操作批量更新所有管理下载器中的 Tracker 地址
- **灵活手动转种**
  即点即转，手动转移种子到任意下载器
  操作直观、无需预设定时任务，作为iyuu-plus的补充功能
- **种子添加 & 文件备份**
  从 BtDeck 直接添加种子到任意下载器
  支持对各下载器种子文件进行备份
- **自定义 Tracker 报错规则**
  可按关键词、错误类型设置忽略规则
  避免把临时限速、维护等误判为严重错误
- **完整操作审计日志**
  所有种子操作（开始/暂停/转移/删除/替换等）均记录时间、类型、来源
  可随时追溯
- **四级安全删除机制**
  1. 彻底删除（种子 + 文件）
  2. 仅删除种子（保留文件）
  3. 回收站删除（BtDeck 暂管，可手动恢复）
  4. 逻辑删除（仅打标签标记待删，目前只支持 qBittorrent 统一核对）

本项目是通过 Claude Code + GLM4.7 进行 vibe engineering 完成的，95%以上的代码均由ai生成，所以可能会出现部分变量、文件命名有问题、代码风格不太一致等。这些问题会随着项目发展逐步解决。

因为教程内容多，为避免写太长，这次就先不写了(~~可以水多一篇~~)。当前项目是作为第一个可用版本，应该还存在大量问题，特别是运行环境，有飞牛、pve、truenas、Windows、Linux、mac等等，虽然理论上docker解决一切但是应该无法覆盖所有疑难杂症，如果有使用上的问题欢迎提issue。

即开即用的docker-compose.yml内容

```
# BTDeck Docker Compose 配置文件
# 包含后端和前端服务的完整部署配置
# 适用于生产环境和开发环境

version: '3.8'

services:
  # ==================== 后端服务 ====================
  # FastAPI + Python 3.11
  # 端口：5001 (API服务)
  backend:
    # 镜像构建配置
    build:
      context: ./
      dockerfile: Dockerfile

    # 镜像名称和容器名称
    image: strainthomas/btdeck-backend:latest
    container_name: btdeck-backend

    # 环境变量配置
    # - PYTHONUNBUFFERED: Python输出不缓冲
    # - TZ: 时区设置
    # - DATABASE_URL: 数据库连接路径
    # - LOG_LEVEL: 日志级别
    environment:
      - PYTHONUNBUFFERED=1
      - TZ=Asia/Shanghai
      - DATABASE_URL=sqlite:///data/app.db
      - LOG_LEVEL=INFO

    # 端口映射：宿主机5001 -> 容器5001，非调试无必要映射开放
    #ports:
    #  - "5001:5001"

    # 数据持久化挂载
    # - data: 应用数据
    # - logs: 日志文件
    # - config: 配置文件
    # - backup: 种子文件备份数据
    volumes:
      - ./data/backend/data:/app/data
      - ./data/backend/logs:/app/logs
      - ./data/backend/config:/app/config
      - ./data/backend/backup:/app/backup

    # 连接到自定义网络
    networks:
      - btdeck_network

    # 资源限制配置
    # - 限制：1核CPU / 1GB内存
    # - 保留：0.25核CPU / 256MB内存
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 1G
        reservations:
          cpus: '0.25'
          memory: 256M

    # 重启策略：除非手动停止，否则自动重启
    restart: unless-stopped

    # 健康检查：每30秒检查API文档页面
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5001/docs"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 30s

  # ==================== 前端服务 ====================
  # Nginx + Vue.js 静态文件
  # 端口：8080 (宿主机) -> 80 (容器)
  frontend:
    # 镜像构建配置
    # 上下文：前端项目根目录
    build:
      context: ../BtDeck_fronted
      dockerfile: Dockerfile.prod

    # 镜像名称和容器名称
    image: strainthomas/btdeck-frontend:latest
    container_name: btdeck-frontend

    environment:
      - TZ=Asia/Shanghai

    # 端口映射：8080作为web访问端口
    ports:
      - "8080:80"

    # 数据持久化挂载
    # - Nginx缓存
    # - Nginx访问日志
    volumes:
      - frontend_cache:/var/cache/nginx
      - frontend_logs:/var/log/nginx

    # 连接到自定义网络
    networks:
      - btdeck_network

    # 资源限制配置
    # - 限制：0.5核CPU / 256MB内存
    # - 保留：0.1核CPU / 128MB内存
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 256M
        reservations:
          cpus: '0.1'
          memory: 128M

    # 重启策略：除非手动停止，否则自动重启
    restart: unless-stopped

    # 健康检查：每30秒检查健康接口
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:80/health"]
      interval: 30s
      timeout: 3s
      retries: 3
      start_period: 10s

    # 依赖后端服务健康检查通过后启动
    depends_on:
      backend:
        condition: service_healthy

# ==================== 网络配置 ====================
# 自定义桥接网络，用于前后端通信
networks:
  btdeck_network:
    driver: bridge

# ==================== 数据卷配置 ====================
# 命名卷：用于前端缓存和日志持久化
volumes:
  frontend_cache:
    driver: local

  frontend_logs:
    driver: local
```
