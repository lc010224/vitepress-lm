# caddy安装

在 Netcup 服务器上使用 Caddy 进行反向代理是一个非常轻量且高效的选择。既然域名（在阿里云）已经解析到了 Netcup 的 IP，我们只需要在服务器上安装 Caddy 并配置 Caddyfile 即可。

## 第一步：在 Netcup 服务器上安装 Caddy
假设你的服务器运行的是 Debian 或 Ubuntu（Netcup 的主流系统），请运行以下命令安装官方版 Caddy：

### 安装必要的依赖

```
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
```

### 添加官方 GPG key
```
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
```

### 添加官方源
```
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
```

### 更新并安装
```
sudo apt update
sudo apt install caddy
```
## 第二步 设置caddyfile
### 打开caddyfile
```
nano /etc/caddy/Caddyfile
```
### 验证配置语法

```
caddy validate -c /etc/caddy/Caddyfile
```
### 重排
```
caddy fmt --overwrite /etc/caddy/Caddyfile
```
### 重启Caddy，触发证书申请
```
systemctl restart caddy
```
