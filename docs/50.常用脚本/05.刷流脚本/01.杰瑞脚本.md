# 杰瑞盒子脚本

## 脚本：

```
bash <(wget -qO- https://raw.githubusercontent.com/jerry048/Dedicated-Seedbox/main/Install.sh) -u <username> -p <password> -c <Cache Size(unit:MiB)> -q <qBittorrent Version> -l <libtorrent Version> -b -v -r -3 -x -o
```

说明：

```
1. -u: username 
2. -p: password
3. -c: Cache size for torrent client
4. -q: qBittorrent versions
5. -l: libtorrent versions
6. -b: Install autobrr
7. -v: Install vertex
8. -r: Install autoremove-torrents
9. -3: Enable BBR V3
10.-x: Enable BBRx
11. Customize ports
```

## 示例：

```

## 🛠️ 性能微调笔记

### 1. 缓存大小 (Cache Size)

- **通用建议**：缓存大小应设置为机器总可用内存的 **1/4** 左右。
- **特殊版本注意**：如果你使用的是 **qBittorrent 4.3.x** 版本，由于存在内存泄漏风险，建议将缓存设置为总内存的 **1/8**。

### 2. 异步 I/O 线程数 (aio_threads)

- **默认值**：通常为 4，这对于机械硬盘（HDD）来说已经足够。
- **固态硬盘（SSD/NVMe）**：如果是 SSD 甚至 NVMe 服务器，建议增加到 **8** 甚至 **16**。
- **设置方法**：
  - **qBittorrent 4.3.x - 4.6.x**：直接在“高级设置”选项卡中修改。
  - **qBittorrent 4.1.x**：需先关闭程序，在配置文件 `/home/$用户名/.config/qBittorrent/qBittorrent.conf` 的 `[BitTorrent]` 节点下添加：`Session\AsyncIOThreadsCount=8`。
  - **Deluge**：安装 `ltconfig` 插件后，在插件中修改 `aio_threads=8`。

### 3. 发送缓冲区水位线 (send_buffer)

涉及参数：`send_buffer_low_watermark`、`send_buffer_watermark` 以及 `send_buffer_watermark_factor`。

- **适用场景**：如果你的机器 **I/O 性能较差**，建议将这些值调低。
- **设置方法**：
  - **qBittorrent 4.3.x**：在“高级设置”中修改。
  - **qBittorrent 4.1.x**：关闭程序后在配置文件 `[BitTorrent]` 节点下添加：`Session\SendBufferWatermark=5120`、`Session\SendBufferLowWatermark=1024` 和 `Session\SendBufferWatermarkFactor=150`。
  - **Deluge**：通过 `ltconfig` 插件修改：
    - `send_buffer_low_watermark=1048576`
    - `send_buffer_watermark=5242880`
    - `send_buffer_watermark_factor=150`

### 4. 勾点间隔 (tick_interval)

- **默认值**：100（单位通常为毫秒）。
- **建议**：对于性能较弱的 CPU，100 可能会导致负载过高。建议根据情况调整为 **250** 或 **500**。
- **限制**：**qBittorrent 无法修改此设置**。
- **Deluge**：通过 `ltconfig` 插件修改 `tick_interval=250`。

### 5. 其他补充

- 更多系统层面的微调建议可以在 `/etc/sysctl.conf` 文件中查看/操作。
- **文件系统推荐**：强烈推荐使用 **XFS** 文件系统，这对处理大文件和高并发 I/O 表现更佳。
