# Markdown 语法速查表

> 说明：本页只演示语法写法，不追求内容含义。你可以在任何文章里复制这些片段，再按需要修改文字。

---

## 1. 标题与段落

### 1.1 标题（Heading）
```md
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```
效果：
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

注意：

- `#` 与标题文字之间要有一个空格。
- 一篇文档建议只用一个一级标题，其余用二级及以下。

### 1.2 段落与换行

```md
这是第一段，中间用一个空行分隔。

这是第二段。

这一行后面(末尾加两个空格)  
强制换行，不需要空一行。
```

## 2. 文本样式

### 2.1 粗体、斜体、删除线

```md
**粗体文本**
*斜体文本*
***粗体+斜体***
~~删除线~~
```
效果：  
**粗体文本**  
*斜体文本*  
***粗体+斜体***  
~~删除线~~

### 2.2 下划线（用 HTML）

```md
<u>带下划线的文本</u>
```
效果：  
<u>带下划线的文本</u>

### 2.3 高亮（Teek/VitePress 推荐用 `==` 或 `<mark>`）

```md
==高亮文本（取决于渲染器是否支持）==

<mark>高亮文本（HTML 标签，兼容性更好）</mark>
```
效果：  
<mark>高亮文本（HTML 标签，兼容性更好）</mark>  

### 2.4 行内代码

```md
请使用 `code` 包裹代码或命令，比如 `npm run dev`。
```

## 3. 列表

### 3.1 无序列表

```md
- 项目 A
- 项目 B
  - 子项目 B1
  - 子项目 B2
* 也可以用 *
+ 或者用 +
```
效果：  
- 项目 A
- 项目 B
  - 子项目 B1
  - 子项目 B2
* 也可以用 *
+ 或者用 +

### 3.2 有序列表

```md
1. 第一项
2. 第二项
3. 第三项
   1. 子项 3.1
   2. 子项 3.2
```

> 注意：数字后必须跟一个英文句点 `.` 和空格。

### 3.3 任务列表（Todo List）

```md
- [ ] 未完成任务
- [x] 已完成任务
- [ ] 支持嵌套
- [ ] 子任务 1
- [x] 子任务 2
```
效果：  
- [ ] 未完成任务
- [x] 已完成任务
- [ ] 支持嵌套
- [ ] 子任务 1
- [x] 子任务 2

## 4. 引用与分割线

### 4.1 引用（Blockquote）

```md
> 这是一个引用块。
> 可以有多行内容。
>
> - 引用中也可以放列表
> - 或者 **加粗文本**
```
效果：  
> 这是一个引用块。
> 可以有多行内容。
>
> - 引用中也可以放列表
> - 或者 **加粗文本**

### 4.2 多级引用

```md
> 一级引用
>> 二级引用
>>> 三级引用
```
效果：  
> 一级引用
>> 二级引用
>>> 三级引用

### 4.3 水平分割线

```md
---
***
___
```

三种写法效果相同：表示一条水平分割线。  
效果：  
---
***
___

效果：  
[显示文字](https://example.com)



```
- `json`
- `html`
- `css` / `scss`

- `yaml` / `yml`
```

## 8. 脚注（Footnote）

> 说明：脚注是否渲染，取决于当前 Markdown 引擎配置。VitePress 默认支持。

```md
这里有一个脚注的例子[^1]。

再来一个脚注[^第二个脚注]。

[^1]: 这里是脚注 1 的内容。
[^第二个脚注]: 这里是脚注 2 的内容。
```
效果：  
这里有一个脚注的例子[^1]。

再来一个脚注[^第二个脚注]。

[^1]: 这里是脚注 1 的内容。
[^第二个脚注]: 这里是脚注 2 的内容。

## 9. 转义与特殊符号

### 9.1 转义特殊符号

以下符号在 Markdown 中有特殊含义，如需原样显示，可以用 `\` 转义：

```md
\* 不会变成列表
\# 不会变成标题
\_ 不会变成斜体
\` 不会变成代码
\| 不会被当作表格分隔符
```

### 9.2 内联 HTML

```md
<span style="color: red;">这是一段红色文字（需要浏览器支持内联样式）。</span>

<details>
  <summary>点击展开详情（原生 HTML 标签）</summary>
  这里是详情内容。
</details>
```

## 13. 文档头部信息（frontmatter）示例

在 VitePress + Teek 中，每个 `.md` 文件顶部的 `---` 部分叫做 frontmatter，用来配置这篇文章的**标题、日期、分类、封面图**等。

```yaml
---
date: 2026-03-13 10:00:00
title: 示例文章标题
permalink: /zhuanti/example-article
categories:
  - 专题
  - 示例分类
tags:
  - 标签1
  - 标签2
coverImg: /img/cover/example.webp
---
```

常见字段说明（会被 Teek 主题用到）：

- `date`：文章创建时间，用于排序和显示。
- `title`：文章标题，不写时默认取文件名。
- `permalink`：生成的固定链接（可选）。
- `categories` / `tags`：分类与标签，用于索引和筛选。
- `coverImg`：文章封面图；如果不写，主题会按配置自动生成。
